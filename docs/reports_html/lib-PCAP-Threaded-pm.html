274         =item _suitable_threads
275        
276         $self->_suitable_threads($divisor);
277        
278         If the code-ref to be executed uses multiple threads (or piped processes) the total number of
279         parallel jobs will be divided by this number to prevent over subscription of resources. Take this
280         simple example:
281        
282         grep -wF 'wibble' | cut -f 5 | sort | uniq -c
283        
284         This 'pipeline' can theoretically use 4 CPU's at 100% each. When executing multiple parallel
285         executions of this the total number of threads defined on the command line is divided by the
286         provided value:
287        
288         max_parallel = int(total_threads / divisor);
289         max_parallel++ if(max_parallel == 0);
290        
291         If total_threads is < divisor a single process will run, any prevention of oversubscription then
292         falls to the OS and any CPU affinity settings.
